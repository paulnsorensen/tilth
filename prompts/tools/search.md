Code search — finds definitions, content, and callers in one default search. **Defaults are tuned — omit `kind` unless you specifically need to filter.** Default merges symbol + content + identifier-shaped callers in one pass; setting `kind` narrows the search to a single mode — faster and smaller for that mode, but you lose the cross-mode results. Prefer omitting `kind` for new queries. Example default query: `tilth_search(queries: [{query: "handleAuth"}])` returns the definition, usages, and call sites together. Use `queries: [{query, glob?, kind?}]` (array form, max 10); per-entry glob/kind compose freely. `kind` values filter to symbol, content, regex, or callers. `expand: N` inlines source for top N matches with `<line>:<3-hex-hash>|<content>` prefixes (same format as tilth_read) — use the `<line>:<3-hex-hash>` prefix (before `|`) as the anchor for tilth_write. If editing `src/edit.rs`, pass `context: "src/edit.rs"` to surface nearby results first. `if_modified_since: <ts>` returns unchanged-file stubs.
